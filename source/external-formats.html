External formats

The native mechanism for the system is the service bundle. However, one can convert and import service configuration from various external formats.

  • generating bundles from systemd units

  • generating bundles from /etc/fstab

  • importing settings from /etc/rcconf

  • importing settings from /etc/ttys

A suite of redo scripts in /etc/system-control/convert/ are supplied for tying these altogether. Simply executing redo all in that directory will attempt to import from various external configuration formats into service bundles. Because this mechanism detects changes to source files, it means that it only performs conversion actions when source configuration files have changed.
